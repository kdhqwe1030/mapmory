# AI Agent Workflow - Mapmory

> 이 파일은 Claude가 매 작업 시작 전 반드시 읽어야 하는 핵심 운영 규칙이다.

---

## 1. 자동 매뉴얼 시스템

### 작업 시작 전 - 매뉴얼 활성화 규칙

아래 조건에 해당하면 `context/` 폴더의 해당 문서를 **반드시 먼저 읽는다**.

| 키워드 / 조건 | 읽어야 할 파일 |
|---|---|
| UI, 컴포넌트, 페이지, 화면, 디자인 | `context/frontend.md` |
| API, 서버, 데이터, DB, 백엔드 | `context/backend.md` |
| 배포, build, CI/CD, 환경변수 | `context/devops.md` |
| 새 기능 추가, 신규 구현 | `context/project.md` + 해당 영역 |
| `src/app/` 파일 수정 | `context/frontend.md` |
| 항상 | `context/project.md` (프로젝트 개요) |

### 작업 시작 전 체크리스트
- [ ] `context/project.md` 읽었는가?
- [ ] 작업 영역에 해당하는 context 파일 읽었는가?
- [ ] `tasks/` 폴더에 현재 진행 중인 태스크가 있는가? (있으면 확인)

## 2. 작업 기억 시스템

큰 작업(기능 추가, 리팩토링 등)을 시작할 때 **반드시 3개 문서를 먼저 만든다**.

### 문서 위치 및 역할

```
.claude/
├── plans/{작업명}.md       # 계획서 - 무엇을 어떻게 만들 것인가 (설계도)
├── context/{주제}.md       # 맥락노트 - 왜 이렇게 결정했는가, 참고 자료 위치
└── tasks/{작업명}.md       # 체크리스트 - 무엇을 끝냈고 무엇이 남았는가
```

### 작업 순서

1. **계획 수립** → AI가 계획 초안 작성
2. **계획 검토** → 사용자가 승인 전 꼼꼼히 검토 (핵심 오해 여부 확인)
3. **문서화** → 승인 즉시 `plans/`, `tasks/` 파일 생성 (코드 작성 전)
4. **작업 실행** → 한 번에 1~2개 작업만 진행
5. **체크리스트 업데이트** → 완료 항목 체크, 다음 항목 확인
6. **새 대화 시작 시** → `plans/`와 `tasks/` 파일 읽고 이어서 진행

### 계획서 템플릿 (`plans/{작업명}.md`)
```markdown
# 계획서: {작업명}
날짜: YYYY-MM-DD

## 목표
무엇을 만드는가

## 접근 방법
어떻게 구현할 것인가

## 단계별 작업
1. ...
2. ...

## 완료 기준
어떤 상태가 되면 완료인가
```

### 체크리스트 템플릿 (`tasks/{작업명}.md`)
```markdown
# 태스크: {작업명}
날짜: YYYY-MM-DD | 상태: 진행중

## 완료
- [x] ...

## 진행중
- [ ] ...

## 대기
- [ ] ...

## 결정 사항 및 메모
- ...
```

## 3. 자동 품질 검사

### 작업 완료 후 셀프 체크 (매번 실행)

코드 수정을 마친 뒤 다음 항목을 스스로 확인한다:

**공통**
- [ ] 수정한 파일에 타입스크립트 오류가 없는가?
- [ ] 콘솔 에러 또는 경고를 남기지 않았는가?
- [ ] 하드코딩된 값이 있지는 않은가?

**UI/프론트엔드**
- [ ] 모바일 반응형을 고려했는가?
- [ ] 접근성(alt, aria 등) 빠뜨린 부분이 없는가?

**로직/데이터**
- [ ] 에러 처리(try-catch, 예외 케이스)를 추가했는가?
- [ ] 보안상 위험한 부분(XSS, SQL Injection 등)은 없는가?

**완료 보고 형식**
작업이 끝나면 아래 형식으로 보고한다:
```
## 완료 보고
- 수정한 파일: ...
- 변경 내용: ...
- 확인한 항목: ...
- 주의사항 또는 다음 단계: ...
```

## 4. 작업 규칙 요약

| 규칙 | 내용 |
|---|---|
| 읽기 먼저 | 코드 수정 전 반드시 파일을 읽는다 |
| 계획 먼저 | 큰 작업은 코드보다 문서를 먼저 만든다 |
| 작게 나누기 | 한 번에 1~2개 작업만 진행한다 |
| 체크리스트 | 완료 후 tasks/ 파일을 업데이트한다 |
| 과잉 금지 | 요청된 것만 한다. 불필요한 파일/추상화 금지 |
| 보고 | 작업 완료 후 수정 내용과 확인 항목을 보고한다 |
